# Академия Claude — интерактивное обучение-игра по всем возможностям Claude

**Статус:** план проекта (v1)
**Дата:** 2026-07-19
**Автор идеи:** zmeyka3777-prog
**Репозиторий:** claude-learning-game (отдельный от Analizator)

---

## 1. Проблема и цель

**Проблема.** Возможностей у Claude очень много (чат, проекты, артефакты, скиллы, Claude Code, MCP, плагины, коннекторы, API…), они быстро растут, документация разрознена и в основном на английском. Большинство пользователей используют 10–20% функционала и даже не знают, что остальное существует.

**Цель.** Создать интерактивное обучение в формате игры, которое:
- проводит человека от «первый раз открыл Claude» до продвинутого уровня (скиллы, MCP, автоматизация);
- учит через практику, а не чтение — «сделай сам» вместо «прочитай»;
- удерживает мотивацию игровыми механиками (XP, уровни, стрики, бейджи);
- на русском языке (потенциально уникальная ниша — качественного игрового обучения Claude на русском нет);
- легко обновляется, когда Anthropic выпускает новые функции.

**Целевые аудитории:**
1. Новички — «что вообще умеет Claude», основы промптов.
2. Активные пользователи claude.ai — проекты, артефакты, скиллы, коннекторы.
3. Разработчики — Claude Code, MCP, API, Agent SDK.
4. Бизнес-пользователи — автоматизация рутины, работа с документами, Excel, Slack.
5. Команды/компании — корпоративный онбординг в Claude (возможная монетизация).

## 2. Полная карта функционала Claude (что покрываем)

Это учебная программа — «карта миров» игры. Каждый мир = крупный блок функционала.

### Мир 1. Основы Claude (новичок)
- Что такое Claude, семейства моделей (Opus, Sonnet, Haiku, Fable/Mythos), чем отличаются, когда какую выбирать.
- Интерфейс claude.ai: чаты, история, поиск по чатам, мобильное приложение, голосовой режим.
- Основы промптинга: контекст, роль, формат ответа, примеры (few-shot), пошаговое рассуждение.
- Загрузка файлов: PDF, изображения, таблицы, код — что Claude умеет с ними делать.
- Ограничения: контекстное окно, дата знаний, галлюцинации и как их проверять.

### Мир 2. Продвинутый claude.ai
- Projects: инструкции проекта, база знаний, когда проект вместо чата.
- Artifacts: интерактивные страницы, приложения, публикация и шаринг.
- Память (memory) и персонализация.
- Deep Research: многошаговое исследование с источниками.
- Коннекторы: Gmail, Google Drive, Calendar, Notion и др. — подключение и сценарии.
- Web search, анализ данных (analysis tool), генерация документов (docx, xlsx, pptx, pdf).

### Мир 3. Скиллы (Skills)
- Что такое скилл, чем отличается от промпта и проекта.
- Использование готовых скиллов, маркетплейс.
- Создание своего скилла: SKILL.md, структура, триггеры-описания.
- skill-creator, тестирование и итерация скиллов.

### Мир 4. Claude Code — база
- Что это, установка, 4 формы: CLI, web (claude.ai/code), desktop, IDE-расширения (VS Code, JetBrains).
- Первая сессия: как ставить задачи, permission modes, что Claude может и не может.
- CLAUDE.md — память проекта, правила, как её строить.
- Slash-команды, ключевые встроенные команды.
- Git-workflow: коммиты, ветки, PR, review.

### Мир 5. Claude Code — продвинутый
- Hooks (PreToolUse, PostToolUse, Stop и др.) — автоматизация вокруг сессий.
- Субагенты (.claude/agents), параллельная работа, worktree-изоляция.
- Собственные slash-команды и скиллы проекта.
- Plan mode, фоновые задачи, cron/routines, мониторинг PR.
- Headless-режим и CI: Claude Code в GitHub Actions.
- Настройки: settings.json, permissions, allowlist.

### Мир 6. MCP и плагины
- Что такое MCP (Model Context Protocol): серверы, тулы, ресурсы.
- Подключение готовых MCP-серверов (GitHub, Figma, Supabase, PostgreSQL…).
- Создание своего MCP-сервера (mcp-builder, FastMCP/TypeScript SDK).
- Плагины Claude Code: маркетплейсы, установка, создание своего плагина.

### Мир 7. API и Agent SDK (для разработчиков)
- Messages API: первый запрос, стриминг, system prompt.
- Tool use: определение инструментов, агентный цикл.
- Prompt caching, батчи, подсчёт токенов, цены и оптимизация.
- Claude Agent SDK: свои агенты на базе движка Claude Code.
- Vision, PDF на входе, structured output.

### Мир 8. Экосистема и интеграции
- Claude in Slack, Claude in Excel, Claude in Chrome, Cowork.
- Claude Design / артефакты как дизайн-инструмент, интеграции Figma/Canva.
- Компьютерное использование (computer use), автоматизация браузера.
- Безопасность: что не хранить в промптах, prompt injection, работа с секретами.

> Карту нужно поддерживать: Anthropic выпускает новое каждые 1–2 месяца. Поэтому контент храним как данные (см. архитектуру), а не хардкодим в компонентах.

## 3. Все возможные форматы игры (варианты)

### Вариант A. «Duolingo-стиль» — карта уровней с короткими уроками
Дорожка миров и уроков, каждый урок 3–7 минут: микро-теория + интерактивные задания. Стрики, XP, лиги.
- **Плюсы:** проверенная механика удержания; идеально для микрообучения; проще всего масштабировать контент.
- **Минусы:** нужно много контента; сама по себе механика не учит «рукам».

### Вариант B. Симулятор-песочница (эмулятор Claude внутри игры)
Встроенные «фейковые» интерфейсы: окно чата claude.ai, терминал Claude Code. Игрок реально печатает команды/промпты, симулятор реагирует по сценарию. Например: «настрой CLAUDE.md», «подключи MCP-сервер», «напиши промпт, чтобы получить JSON».
- **Плюсы:** максимальное обучение через практику, безопасно (ничего не сломаешь), не требует подписки Claude.
- **Минусы:** дорогая разработка сценариев; симуляция всегда беднее реальности.

### Вариант C. Реальные миссии (задания в настоящем Claude)
Игра выдаёт миссию («создай артефакт-калькулятор», «сделай скилл и запусти»), игрок выполняет в реальном Claude и вставляет результат/скриншот/ссылку; проверка — чек-лист самопроверки или проверка через API.
- **Плюсы:** реальный навык, нулевая стоимость симуляции.
- **Минусы:** нужен аккаунт Claude; автопроверка сложна; разрыв «вышел из игры — не вернулся».

### Вариант D. RPG с прокачкой персонажа
Персонаж «AI-инженер», дерево навыков (ветки: Промптер / Разработчик / Автоматизатор), квесты, экипировка = разблокированные инструменты, боссы = итоговые проекты.
- **Плюсы:** сильная нарративная мотивация, вау-эффект.
- **Минусы:** самая дорогая разработка; сюжет вторичен к обучению.

### Вариант E. Квиз/викторина
Вопросы с вариантами, таймер, дуэли с другими игроками, таблица лидеров. Быстро сделать, подходит как один из типов заданий, но не как весь продукт.

### Вариант F. Эскейп-рум / детектив
Цепочка загадок: «чтобы открыть дверь, составь правильный промпт», «найди ошибку в конфиге MCP». Отлично работает как спецуровни-боссы внутри других форматов.

### Вариант G. Claude как гейм-мастер (мета-вариант!)
Обучение ведёт сам Claude: скилл или проект «Наставник по Claude», который ведёт диалоговую игру, даёт задания, проверяет ответы, адаптируется под уровень. Распространение — просто ссылка/скилл, разработки почти нет.
- **Плюсы:** делается за дни; обучение внутри самого инструмента; самопроверка настоящим Claude.
- **Минусы:** нет визуальной игры, прогресс-механики ограничены, нужна подписка.

### Вариант H. Telegram-бот
Ежедневные микро-уроки и квизы в Telegram, стрики, рейтинг. Идеален как канал вовлечения/напоминаний, дополняющий веб-игру.

### Вариант I. Коллекционные карточки
Каждая изученная функция = карточка (редкость по глубине функции). Альбом коллекции, обмен. Хорошая мета-механика поверх любого варианта.

### Вариант J. Сертификация
Финальные экзамены по трекам, сертификат (страница с проверяемой ссылкой). Мотивация для профессиональной аудитории и основа B2B-монетизации.

## 4. Рекомендуемая концепция: «Академия Claude» (гибрид A + B + C + I + J)

Веб-приложение (SPA) с:
1. **Картой миров** в стиле Duolingo (вариант A) — 8 миров из раздела 2, внутри миров уроки.
2. **Уроки из микро-теории + интерактивных заданий** 6 типов:
   - квиз (один/несколько ответов);
   - «собери промпт» — drag-and-drop блоков в правильный промпт;
   - «найди ошибку» — в промпте/конфиге/команде;
   - симулятор чата — мини-сценарий диалога с «Claude» (вариант B);
   - симулятор терминала — ввод команд Claude Code с проверкой (вариант B);
   - реальная миссия — задание в настоящем Claude с чек-листом (вариант C), даёт больше всего XP.
3. **Геймификация:** XP, уровни, стрики, бейджи-достижения, карточки функций (вариант I), боссы-уровни в конце мира (вариант F как формат босса).
4. **Треки** (роли при онбординге): «Новичок», «Пользователь claude.ai», «Разработчик», «Бизнес» — трек определяет порядок и обязательность миров.
5. **Сертификат** по завершении трека (вариант J) — на потом.
6. Параллельно (дёшево, быстро) можно выпустить **вариант G** — скилл «Наставник по Claude» как ранний продукт и источник обратной связи, пока строится веб-игра.

Почему так: A даёт удержание и структуру, B — практику без барьера подписки, C — настоящий навык, I/F/J — глубину мотивации. Всё это ложится на один контентный движок.

## 5. Технологии

Максимально переиспользуем знакомый стек (тот же, что в Analizator — меньше рисков):

| Слой | Выбор | Комментарий |
|---|---|---|
| Frontend | React 18 + TypeScript + Vite | знакомо, быстро |
| Стили | Tailwind CSS 4 | знакомо |
| UI | shadcn/ui + Lucide | знакомо |
| Анимации | Motion (Framer Motion) | карта миров, награды, конфетти |
| Состояние | Zustand + localStorage | прогресс без бэкенда в MVP |
| Контент | JSON/Markdown-файлы в `content/` | уроки = данные, не код |
| Симулятор терминала | xterm.js или самописный | ввод команд по сценарию |
| Drag-and-drop | dnd-kit | задания «собери промпт» |
| Бэкенд (фаза 2+) | Supabase (auth + PostgreSQL) | аккаунты, лидерборды, синхронизация прогресса |
| Деплой | Vercel или GitHub Pages | MVP полностью статический |

Принцип MVP: **ноль бэкенда**. Весь прогресс в localStorage, приложение — статический сайт. Supabase добавляем только когда понадобятся аккаунты и лидерборды.

## 6. Архитектура

```
claude-learning-game/
├── docs/
│   └── PLAN.md                  # этот документ
├── content/                     # ВЕСЬ учебный контент = данные
│   ├── worlds.json              # миры: id, название, порядок, иконка, трек
│   ├── world-1-basics/
│   │   ├── lesson-1-what-is-claude.json
│   │   ├── lesson-2-first-prompt.json
│   │   └── ...
│   ├── world-4-claude-code/
│   │   └── ...
│   ├── badges.json              # достижения
│   └── cards.json               # коллекционные карточки функций
├── src/
│   ├── engine/                  # движок уроков (не знает о конкретном контенте)
│   │   ├── types.ts             # Lesson, Task, TaskType, Progress...
│   │   ├── lessonRunner.ts      # проход по заданиям, подсчёт XP
│   │   ├── progressStore.ts     # Zustand + localStorage
│   │   └── validators/          # проверка ответов по типам заданий
│   ├── components/
│   │   ├── map/                 # карта миров
│   │   ├── lesson/              # оболочка урока, прогресс-бар
│   │   ├── tasks/               # QuizTask, BuildPromptTask, FindBugTask,
│   │   │                        # ChatSimTask, TerminalSimTask, RealMissionTask
│   │   ├── gamification/        # XP-бар, стрик, бейджи, карточки, конфетти
│   │   └── ui/                  # shadcn
│   ├── pages/                   # Map, Lesson, Profile, Badges, Cards, About
│   └── App.tsx
└── package.json
```

**Ключевое решение:** урок — это JSON-документ, который движок умеет проиграть. Добавление нового урока или обновление под новую функцию Claude = добавить/поправить JSON, без кода. Пример схемы урока:

```json
{
  "id": "cc-claude-md",
  "world": "world-4-claude-code",
  "title": "CLAUDE.md — память твоего проекта",
  "xp": 50,
  "theory": [
    { "type": "text", "md": "CLAUDE.md — файл в корне репозитория..." },
    { "type": "code", "lang": "markdown", "code": "# CLAUDE.md\n## Команды\nnpm run dev" }
  ],
  "tasks": [
    { "type": "quiz", "question": "Где Claude Code ищет CLAUDE.md?", "options": ["..."], "correct": [0, 2] },
    { "type": "find-bug", "code": "...", "bugLine": 3, "explanation": "..." },
    { "type": "terminal-sim", "scenario": "claude-md-init", "goal": "Создай CLAUDE.md командой /init" },
    { "type": "real-mission", "instructions": "Открой свой проект в Claude Code и...", "checklist": ["...", "..."] }
  ],
  "reward": { "card": "card-claude-md", "badge": null }
}
```

**Симуляторы** — сценарные конечные автоматы: сценарий описывает ожидаемые вводы (regex/наборы вариантов), ответы «Claude»/терминала, ветки при ошибках и подсказки после N неудач.

## 7. Геймификация — детально

- **XP и уровни:** задания дают 5–20 XP, реальные миссии 50–100. Уровни по нарастающей шкале, уровень отображается в профиле.
- **Стрик:** дни подряд с ≥1 уроком; заморозка стрика как награда за бейджи.
- **Бейджи** (~30 на старте): «Первый промпт», «Архитектор проекта» (мир 2), «Повелитель терминала» (мир 4), «MCP-инженер» (мир 6), «Марафонец» (стрик 30) и т.д.
- **Карточки функций:** каждая функция Claude = карточка с редкостью (обычная: web search → легендарная: свой MCP-сервер). Альбом коллекции — наглядная «карта того, что ты уже знаешь и чего не знаешь» — это само по себе отвечает на исходную боль «не знают полный функционал».
- **Боссы миров:** финальный уровень мира — комплексный сценарий-эскейпрум без подсказок.
- **Лидерборды и лиги** — только после появления бэкенда (фаза 3).

## 8. Дорожная карта

### Фаза 0 — Контент-скелет (1–2 недели работы)
- [ ] Утвердить структуру миров и список уроков (полный список из раздела 2 → ~60–80 уроков).
- [ ] Написать схему JSON урока (типы TypeScript).
- [ ] Написать контент мира 1 полностью (8–10 уроков) как эталон.
- [ ] Прототип скилла «Наставник по Claude» (вариант G) — быстрый выпуск для проверки спроса.

### Фаза 1 — MVP веб-игры (3–4 недели)
- [ ] Каркас: Vite + React + TS + Tailwind + shadcn, роутинг, Zustand.
- [ ] Движок уроков + 3 типа заданий: квиз, «собери промпт», «найди ошибку».
- [ ] Карта миров (пока миры 1–2), экран урока, XP, стрик, localStorage-прогресс.
- [ ] Профиль и экран бейджей (5–10 бейджей).
- [ ] Деплой на Vercel/GitHub Pages.
- **Критерий готовности:** незнакомый человек проходит мир 1 без объяснений и узнаёт ≥3 новые функции Claude.

### Фаза 2 — Симуляторы и практика (3–4 недели)
- [ ] Симулятор чата claude.ai (сценарные диалоги).
- [ ] Симулятор терминала Claude Code.
- [ ] Тип «реальная миссия» с чек-листами.
- [ ] Контент миров 3–5 (скиллы, Claude Code база и продвинутый).
- [ ] Карточки функций и альбом.

### Фаза 3 — Полный контент и соцмеханики (4+ недели)
- [ ] Миры 6–8 (MCP, API, экосистема).
- [ ] Боссы миров, треки по ролям при онбординге.
- [ ] Supabase: аккаунты, синхронизация прогресса, лидерборды, лиги.
- [ ] Telegram-бот напоминаний/микроквизов (вариант H).

### Фаза 4 — Рост (по результатам)
- [ ] Сертификаты по трекам.
- [ ] Английская версия (i18n заложить с фазы 1: все строки в контенте, не в коде).
- [ ] B2B-режим: корпоративный онбординг команд, приватные лидерборды.
- [ ] Пользовательский контент: конструктор уроков.

## 9. Поддержание актуальности (критично!)

Claude обновляется быстрее, чем любой курс. Меры:
1. Контент = данные: обновление урока не требует релиза кода.
2. Поле `verifiedAt` в каждом уроке; ежемесячная ревизия по release notes Anthropic.
3. Раздел «Что нового» в игре — свежие функции как спецуроки, это же повод вернуться (ре-энгейджмент).
4. Использовать Claude Code для полуавтоматической проверки контента: сессия раз в месяц «сверь уроки с текущей документацией и предложи правки».

## 10. Риски

| Риск | Митигция |
|---|---|
| Контента слишком много, не хватит сил | MVP = 2 мира; движок позволяет наращивать постепенно |
| Симуляторы дороги в разработке | В MVP их нет; сценарии добавляются по одному |
| Функции Claude устаревают | Раздел 9 |
| Товарные знаки Anthropic | Название без слова «Claude» как бренда продукта, явный дисклеймер «неофициальный проект»; изучить brand guidelines Anthropic |
