# Мультиязычность (i18n) — соглашение

Игра запускается на русском, добавляется английский, архитектура — под любые языки.

## Языки

- Коды: `ru` (базовый), `en`. Расширяется добавлением кода в `LANGUAGES` и файлов перевода.
- Выбранный язык хранится в progress store: поле `lang: LangCode` (по умолчанию `ru`),
  экшен `setLang`. Персистится в localStorage вместе с прогрессом.
- Переключатель — в верхней панели (топбар) рядом с профилем: компактный тумблер RU/EN
  (расширяемый в выпадающий список при >2 языках). Плюс шаг выбора языка в самом начале онбординга.

## UI-строки (интерфейс)

- Все интерфейсные строки (кнопки, заголовки страниц, подписи, тосты) — в словаре
  `src/i18n/strings.ts`: `export const STRINGS: Record<LangCode, Record<string, string>>`.
- Доступ через хук `useT()` → функция `t('key')`, берёт строку активного языка,
  фолбэк на `ru`, затем на сам ключ. Поддержка простых плейсхолдеров `{n}`.
- Никаких «зашитых» русских строк в компонентах после миграции.

## Контент (уроки, миры, карточки и т.д.)

Английский контент — **зеркало** русского под префиксом `content/en/`, те же относительные
пути, те же `id`, английский текст. Русские файлы остаются на своих местах (это язык `ru`).

| Русский (базовый) | Английский |
|---|---|
| `content/worlds.json` | `content/en/worlds.json` |
| `content/cards.json` | `content/en/cards.json` |
| `content/badges.json` | `content/en/badges.json` |
| `content/placement.json` | `content/en/placement.json` |
| `content/changelog.json` | `content/en/changelog.json` |
| `content/library/skills.json` | `content/en/library/skills.json` |
| `content/library/plugins.json` | `content/en/library/plugins.json` |
| `content/library/mcp.json` | `content/en/library/mcp.json` |
| `content/world-1-basics/basics-first-prompt.json` | `content/en/world-1-basics/basics-first-prompt.json` |
| … все уроки всех миров | `content/en/…` тем же путём |

### Правила перевода контента

- **`id`, `world`, `order`, `xp`, `durationMin`, `isBoss`, `reward.cardId`, `reward.badgeId`,
  `sources[].url`, все технические поля, `correct`/`correctOrder`/`bugLineIndex`/`expectPattern`/
  `minMatches` — НЕ меняются.** Совпадение `id` критично: движок сопоставляет ru/en по нему.
- Переводится только человекочитаемый текст: `title`, `subtitle`, `theory[].*` (md/text/code-comments/
  callout/table/example), `tasks[].*` (question, options, explanation, hint, instruction, blocks,
  distractors, claudeReply, failHint, successMessage, checklist, goal), `sources[].title`,
  `category`/`description`/`useFor`/`install`-подписи в библиотеке (сами команды установки не трогать),
  `name` карточек/миров, `rarity` НЕ переводить (это ключ).
- В `chat-sim`/`terminal-sim`: `expect`/`expectPattern` — это то, что ВВОДИТ игрок, поэтому в английской
  версии переведи ожидаемые слова на английский (игрок будет печатать по-английски). `output`/`claudeReply`
  перевести. Для terminal-команд (`claude`, `/init`) паттерн оставить как есть.
- Термины Claude — официальные английские (Projects, Artifacts, Skills, Claude Code, hooks, MCP…).
- Тон: тот же дружелюбный наставник на «you».

## Загрузчик контента (content.ts)

- Локаль-независимый разбор: строить два индекса по `id` — базовый (`ru`) и `en` (пути `content/en/…`).
- Публичные геттеры принимают активный язык (или читают его из lang-хранилища): для `en` берут
  запись из en-индекса, при отсутствии — фолбэк на `ru` (частичный перевод не ломает игру).
- `import.meta.glob('/content/**/*.json')` уже захватывает и `content/en/**` — надо просто разнести по
  локали по префиксу пути, не сломав существующую загрузку `ru`.

## Порядок внедрения

1. Инфраструктура: lang-store, словарь UI, `useT`, тумблер, шаг языка в онбординге, локаль-загрузчик — с фолбэком на ru.
2. Перевод контента по мирам (можно частично — фолбэк покрывает непереведённое).
3. Еженедельная рутина: при добавлении/правке ru-контента — сразу переводить в `content/en/`.
